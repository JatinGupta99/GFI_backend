# Requirements Document

## Introduction

The ForeSight PDF Extractor is a feature that extracts structured financial data from ForeSight Detail Proforma PDF documents. The system processes PDFs containing property, suite, and tenant financial information, extracting monthly charges, payments, and lease terms into a standardized JSON format. This enables automated processing of financial data that would otherwise require manual data entry.

## Glossary

- **ForeSight_PDF**: A Detail Proforma PDF document generated by the ForeSight system containing financial data
- **Property**: A real estate property identified by a numeric Property ID
- **Suite**: A rentable unit within a property, identified by a Suite ID
- **Tenant**: An entity leasing a suite (implicit in the Property → Suite → Tenant hierarchy)
- **BRR**: Base Rent Recovery - the base rental income row in the PDF
- **CAM**: Common Area Maintenance - recoverable expenses for common areas
- **INS**: Insurance Recovery - recoverable insurance expenses
- **RET**: Real Estate Tax Recovery - recoverable property tax expenses
- **Monthly_PMT**: Monthly Payment calculated as BRR + CAM + INS + RET
- **Total_Due_Month**: Total amount due per month calculated as Base Rent + CAM + INS + TAX
- **Extractor**: The PDF extraction system
- **Extraction_Log**: A record describing how values were extracted and calculated

## Requirements

### Requirement 1: Parse ForeSight PDF Documents

**User Story:** As a financial analyst, I want to parse ForeSight Detail Proforma PDFs, so that I can access the raw text content for data extraction.

#### Acceptance Criteria

1. WHEN a ForeSight_PDF is provided, THE Extractor SHALL parse the PDF into text content
2. WHEN the PDF cannot be parsed, THE Extractor SHALL return a descriptive error message
3. WHEN the PDF is successfully parsed, THE Extractor SHALL preserve the text structure for pattern matching

### Requirement 2: Extract Property Information

**User Story:** As a financial analyst, I want to extract property-level information, so that I can identify which property the financial data belongs to.

#### Acceptance Criteria

1. WHEN parsing a ForeSight_PDF, THE Extractor SHALL extract the Property ID from suite identifiers
2. WHEN parsing a ForeSight_PDF, THE Extractor SHALL extract the property name
3. WHEN parsing a ForeSight_PDF, THE Extractor SHALL extract the region code
4. WHEN multiple suites exist for the same property, THE Extractor SHALL use the same Property ID for all suites

### Requirement 3: Extract Suite Identifiers

**User Story:** As a financial analyst, I want to extract suite identifiers, so that I can associate financial data with specific suites.

#### Acceptance Criteria

1. WHEN a suite identifier is found in format "PPPPPP-SSS", THE Extractor SHALL extract PPPPPP as the Property ID
2. WHEN a suite identifier is found in format "PPPPPP-SSS", THE Extractor SHALL extract SSS as the Suite ID
3. WHEN a suite identifier does not match the expected format, THE Extractor SHALL log an error and skip that suite

### Requirement 4: Extract Base Rent Data

**User Story:** As a financial analyst, I want to extract base rent amounts, so that I can calculate total monthly charges.

#### Acceptance Criteria

1. WHEN a "Rental Income BRR" row is found for a suite, THE Extractor SHALL extract the monthly base rent amount
2. WHEN extracting base rent, THE Extractor SHALL remove currency symbols and commas
3. WHEN extracting base rent, THE Extractor SHALL return a numeric value
4. WHEN no base rent is found for a suite, THE Extractor SHALL return 0

### Requirement 5: Extract CAM Recovery Data

**User Story:** As a financial analyst, I want to extract CAM recovery amounts, so that I can calculate total monthly charges.

#### Acceptance Criteria

1. WHEN a "CAM Recovery - Billed" row is found for a suite, THE Extractor SHALL extract the monthly CAM amount
2. WHEN extracting CAM, THE Extractor SHALL remove currency symbols and commas
3. WHEN extracting CAM, THE Extractor SHALL return a numeric value
4. WHEN no CAM is found for a suite, THE Extractor SHALL return 0

### Requirement 6: Extract Insurance Recovery Data

**User Story:** As a financial analyst, I want to extract insurance recovery amounts, so that I can calculate total monthly charges.

#### Acceptance Criteria

1. WHEN an "INS Recovery - Billed" row is found for a suite, THE Extractor SHALL extract the monthly insurance amount
2. WHEN extracting insurance, THE Extractor SHALL remove currency symbols and commas
3. WHEN extracting insurance, THE Extractor SHALL return a numeric value
4. WHEN no insurance is found for a suite, THE Extractor SHALL return 0

### Requirement 7: Extract Tax Recovery Data

**User Story:** As a financial analyst, I want to extract tax recovery amounts, so that I can calculate total monthly charges.

#### Acceptance Criteria

1. WHEN a "RET Recovery - Billed" row is found for a suite, THE Extractor SHALL extract the monthly tax amount
2. WHEN extracting tax, THE Extractor SHALL remove currency symbols and commas
3. WHEN extracting tax, THE Extractor SHALL return a numeric value
4. WHEN no tax is found for a suite, THE Extractor SHALL return 0

### Requirement 8: Extract Monthly Payment Data

**User Story:** As a financial analyst, I want to extract monthly payment amounts for all 12 months, so that I can track payment history.

#### Acceptance Criteria

1. WHEN extracting monthly payments, THE Extractor SHALL extract values for all 12 months (Jan through Dec)
2. WHEN extracting monthly payments, THE Extractor SHALL remove currency symbols and commas
3. WHEN extracting monthly payments, THE Extractor SHALL return numeric values
4. WHEN a monthly payment is missing, THE Extractor SHALL return 0 for that month
5. FOR ALL monthly payments, THE Extractor SHALL calculate the value as BRR + CAM + INS + RET for that month

### Requirement 9: Calculate Total Due Per Month

**User Story:** As a financial analyst, I want the system to calculate total due per month, so that I can verify the total charges.

#### Acceptance Criteria

1. FOR ALL suites, THE Extractor SHALL calculate Total_Due_Month as Base Rent + CAM + INS + TAX
2. WHEN calculating Total_Due_Month, THE Extractor SHALL return a numeric value
3. WHEN any component is missing, THE Extractor SHALL treat it as 0 in the calculation

### Requirement 10: Handle Missing Lease Terms

**User Story:** As a financial analyst, I want the system to handle missing lease term fields, so that the extraction doesn't fail when optional data is absent.

#### Acceptance Criteria

1. WHEN Balance Due is not found, THE Extractor SHALL return 0
2. WHEN Rent Due Date is not found, THE Extractor SHALL return null
3. WHEN Late After is not found, THE Extractor SHALL return null
4. WHEN Late Fee is not found, THE Extractor SHALL return 0

### Requirement 11: Generate Structured JSON Output

**User Story:** As a financial analyst, I want the extracted data in JSON format, so that I can integrate it with other systems.

#### Acceptance Criteria

1. WHEN extraction is complete, THE Extractor SHALL generate a JSON object with propertyId, propertyName, region, and suites array
2. FOR ALL suites, THE Extractor SHALL include suiteId, charges, balanceDue, leaseTerms, and monthlyPayments
3. WHEN generating JSON, THE Extractor SHALL include createdAt and updatedAt timestamps in ISO 8601 format
4. WHEN generating JSON, THE Extractor SHALL ensure all numeric fields contain numbers, not strings

### Requirement 12: Generate Extraction Logs

**User Story:** As a financial analyst, I want extraction logs, so that I can understand how values were extracted and verify accuracy.

#### Acceptance Criteria

1. FOR ALL extracted values, THE Extractor SHALL generate an Extraction_Log entry describing the extraction method
2. FOR ALL calculated values, THE Extractor SHALL generate an Extraction_Log entry showing the calculation formula
3. WHEN extraction fails for a field, THE Extractor SHALL log the failure reason
4. WHEN extraction completes, THE Extractor SHALL include all logs in the output

### Requirement 13: Validate Extracted Data

**User Story:** As a financial analyst, I want the system to validate extracted data, so that I can trust the accuracy of the results.

#### Acceptance Criteria

1. WHEN extracting numeric values, THE Extractor SHALL validate that the result is a valid number
2. WHEN a suite identifier is malformed, THE Extractor SHALL log a validation error
3. WHEN required fields are missing, THE Extractor SHALL log a validation warning
4. WHEN validation errors occur, THE Extractor SHALL include them in the Extraction_Log

### Requirement 14: Provide HTTP API Endpoint

**User Story:** As a developer, I want an HTTP API endpoint, so that I can integrate PDF extraction into my application.

#### Acceptance Criteria

1. THE Extractor SHALL provide a POST endpoint for PDF upload
2. WHEN a PDF is uploaded, THE Extractor SHALL return the extracted JSON data
3. WHEN extraction fails, THE Extractor SHALL return an appropriate HTTP error status
4. WHEN extraction succeeds, THE Extractor SHALL return HTTP 200 with the JSON payload
