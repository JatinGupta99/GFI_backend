# Requirements Document

## Introduction

The Excel Budget Parser is a feature that extracts structured tenant data from ForeSight Detail Proforma Excel reports. The system processes Excel files containing property and tenant financial information spread across multiple sections, extracting monthly charges, payments, and lease terms into a standardized JSON format. This enables automated processing of budget data from Excel files that would otherwise require manual data entry.

## Glossary

- **ForeSight_Excel**: A Detail Proforma Excel document generated by the ForeSight system containing budget data
- **Property**: A real estate property identified by a numeric Property ID
- **Suite**: A rentable suite within a property, identified by a Suite ID
- **Tenant**: An entity leasing a suite, can be "Proposed" (vacant) or actual tenant name
- **Base_Rent**: Monthly base rental income for a suite
- **CAM**: Common Area Maintenance - recoverable expenses for common areas
- **INS**: Insurance Recovery - recoverable insurance expenses
- **RET**: Real Estate Tax Recovery - recoverable property tax expenses
- **TI**: Tenant Improvements - capital expenditures for tenant buildout
- **Monthly_Payments**: Payment amounts for all 12 months from Leasing Commission section
- **Parser**: The Excel extraction system
- **Suite_Data**: Complete tenant data structure for a single suite

## Requirements

### Requirement 1: Parse ForeSight Excel Documents

**User Story:** As a financial analyst, I want to parse ForeSight Detail Proforma Excel files, so that I can access the structured data for tenant extraction.

#### Acceptance Criteria

1. WHEN a ForeSight_Excel file is provided, THE Parser SHALL read the Excel workbook structure
2. WHEN the Excel file cannot be parsed, THE Parser SHALL return a descriptive error message
3. WHEN the Excel file is successfully parsed, THE Parser SHALL preserve cell values and positions for data extraction
4. WHEN the Excel file is malformed or corrupted, THE Parser SHALL handle the error gracefully

### Requirement 2: Extract Suite Identifiers from Tenant Lines

**User Story:** As a financial analyst, I want to extract suite and property identifiers from tenant line items, so that I can associate financial data with specific suites.

#### Acceptance Criteria

1. WHEN a tenant line contains format "SSSS - Proposed (BRR) PPPPPP-SSSS", THE Parser SHALL extract SSSS as the Suite ID
2. WHEN a tenant line contains format "SSSS - Proposed (BRR) PPPPPP-SSSS", THE Parser SHALL extract PPPPPP as the Property ID
3. WHEN a tenant line contains an actual tenant name instead of "Proposed", THE Parser SHALL extract the Suite ID and Property ID using the same pattern
4. WHEN a tenant line does not match expected formats, THE Parser SHALL log an error and skip that tenant

### Requirement 3: Extract Square Footage Data

**User Story:** As a financial analyst, I want to extract square footage for each suite, so that I can calculate per-square-foot metrics.

#### Acceptance Criteria

1. WHEN processing a tenant in the Rental Income section, THE Parser SHALL extract the square footage from the end of the tenant row
2. WHEN extracting square footage, THE Parser SHALL return a numeric value
3. WHEN square footage is not found for a tenant, THE Parser SHALL log a warning and return 0
4. WHEN square footage contains non-numeric characters, THE Parser SHALL clean and convert to numeric

### Requirement 4: Extract Base Rent from Rental Income Section

**User Story:** As a financial analyst, I want to extract monthly base rent amounts, so that I can calculate total charges and per-square-foot rates.

#### Acceptance Criteria

1. WHEN processing the Rental Income section, THE Parser SHALL extract the monthly base rent value for each tenant
2. WHEN extracting base rent, THE Parser SHALL remove currency symbols and commas
3. WHEN extracting base rent, THE Parser SHALL return a numeric value
4. WHEN no base rent is found for a tenant, THE Parser SHALL return 0

### Requirement 5: Calculate Base Rent Per Square Foot

**User Story:** As a financial analyst, I want the system to calculate base rent per square foot, so that I can compare rental rates across suites.

#### Acceptance Criteria

1. FOR ALL suites with valid square footage, THE Parser SHALL calculate baseRentPerSf as (baseRentMonth × 12) / squareFootage
2. WHEN calculating baseRentPerSf, THE Parser SHALL return a numeric value rounded to 2 decimal places
3. WHEN square footage is 0 or missing, THE Parser SHALL return 0 for baseRentPerSf
4. WHEN base rent is 0, THE Parser SHALL return 0 for baseRentPerSf

### Requirement 6: Extract CAM Recovery Data

**User Story:** As a financial analyst, I want to extract CAM recovery amounts from the CAM Recovery - Billed section, so that I can calculate total monthly charges.

#### Acceptance Criteria

1. WHEN processing the "CAM Recovery – Billed" section, THE Parser SHALL extract the monthly CAM amount for each tenant
2. WHEN extracting CAM, THE Parser SHALL remove currency symbols and commas
3. WHEN extracting CAM, THE Parser SHALL return a numeric value
4. WHEN no CAM is found for a tenant, THE Parser SHALL return 0

### Requirement 7: Extract Insurance Recovery Data

**User Story:** As a financial analyst, I want to extract insurance recovery amounts from the INS Recovery - Billed section, so that I can calculate total monthly charges.

#### Acceptance Criteria

1. WHEN processing the "INS Recovery – Billed" section, THE Parser SHALL extract the monthly insurance amount for each tenant
2. WHEN extracting insurance, THE Parser SHALL remove currency symbols and commas
3. WHEN extracting insurance, THE Parser SHALL return a numeric value
4. WHEN no insurance is found for a tenant, THE Parser SHALL return 0

### Requirement 8: Extract Tax Recovery Data

**User Story:** As a financial analyst, I want to extract tax recovery amounts from the RET Recovery - Billed section, so that I can calculate total monthly charges.

#### Acceptance Criteria

1. WHEN processing the "RET Recovery – Billed" section, THE Parser SHALL extract the monthly tax amount for each tenant
2. WHEN extracting tax, THE Parser SHALL remove currency symbols and commas
3. WHEN extracting tax, THE Parser SHALL return a numeric value
4. WHEN no tax is found for a tenant, THE Parser SHALL return 0

### Requirement 9: Calculate Total Due Per Month

**User Story:** As a financial analyst, I want the system to calculate total due per month, so that I can verify the total charges for each tenant.

#### Acceptance Criteria

1. FOR ALL tenants, THE Parser SHALL calculate totalDueMonth as baseRentMonth + camMonth + insMonth + taxMonth
2. WHEN calculating totalDueMonth, THE Parser SHALL return a numeric value
3. WHEN any component is missing, THE Parser SHALL treat it as 0 in the calculation
4. WHEN all components are 0, THE Parser SHALL return 0 for totalDueMonth

### Requirement 10: Extract Tenant Improvement Data

**User Story:** As a financial analyst, I want to extract tenant improvement costs, so that I can calculate TI per square foot.

#### Acceptance Criteria

1. WHEN processing the "Tenant Improvements" section, THE Parser SHALL extract the TI Amount for each tenant
2. WHEN extracting TI Amount, THE Parser SHALL remove currency symbols and commas
3. FOR ALL tenants with valid TI Amount and square footage, THE Parser SHALL calculate tiPerSf as TI Amount / squareFootage
4. WHEN TI Amount or square footage is 0 or missing, THE Parser SHALL return 0 for tiPerSf

### Requirement 11: Extract Monthly Payment Data

**User Story:** As a financial analyst, I want to extract monthly payment amounts for all 12 months from the Leasing Commission section, so that I can track payment schedules.

#### Acceptance Criteria

1. WHEN processing the "Leasing Commission" section, THE Parser SHALL extract payment values for all 12 months (Jan through Dec) for each tenant
2. WHEN extracting monthly payments, THE Parser SHALL remove currency symbols and commas
3. WHEN extracting monthly payments, THE Parser SHALL return numeric values for each month
4. WHEN a monthly payment is missing or empty, THE Parser SHALL return 0 for that month
5. THE Parser SHALL structure monthly payments as an object with keys: jan, feb, mar, apr, may, jun, jul, aug, sep, oct, nov, dec

### Requirement 12: Determine Tenant Status

**User Story:** As a financial analyst, I want the system to determine tenant status, so that I can identify vacant vs occupied suites.

#### Acceptance Criteria

1. WHEN a tenant line contains "Proposed", THE Parser SHALL set status to "Vacant"
2. WHEN a tenant line contains an actual tenant name (not "Proposed"), THE Parser SHALL set status to "Occupied"
3. WHEN tenant status cannot be determined, THE Parser SHALL set status to "Unknown"
4. THE Parser SHALL include the status field in the Suite_Data output

### Requirement 13: Handle Multiple Tenants Per Property

**User Story:** As a financial analyst, I want the system to process multiple tenants within the same property, so that I can extract data for all suites in a building.

#### Acceptance Criteria

1. WHEN an Excel file contains multiple tenants for the same property, THE Parser SHALL extract data for each tenant separately
2. WHEN processing multiple tenants, THE Parser SHALL maintain the same Property ID for all tenants in the same property
3. WHEN processing multiple tenants, THE Parser SHALL ensure each tenant has a unique Suite ID
4. WHEN duplicate Suite IDs are found, THE Parser SHALL log a warning and process both entries

### Requirement 14: Generate Structured JSON Output

**User Story:** As a financial analyst, I want the extracted data in the specified JSON format, so that I can integrate it with the existing suites system.

#### Acceptance Criteria

1. FOR ALL extracted tenants, THE Parser SHALL generate a JSON object with propertyId, suiteId, status, squareFootage, baseRentMonth, baseRentPerSf, camMonth, insMonth, taxMonth, totalDueMonth, tiPerSf, and monthlyPayments
2. WHEN generating JSON, THE Parser SHALL ensure all numeric fields contain numbers, not strings
3. WHEN generating JSON, THE Parser SHALL ensure monthlyPayments is an object with 12 month keys
4. WHEN generating JSON, THE Parser SHALL return an array of Suite_Data objects for multiple tenants

### Requirement 15: Validate Extracted Data

**User Story:** As a financial analyst, I want the system to validate extracted data, so that I can trust the accuracy of the results.

#### Acceptance Criteria

1. WHEN extracting numeric values, THE Parser SHALL validate that the result is a valid number
2. WHEN calculated values are outside expected ranges, THE Parser SHALL log a validation warning
3. WHEN required fields (propertyId, suiteId) are missing, THE Parser SHALL log a validation error
4. WHEN validation errors occur, THE Parser SHALL include error details in the response

### Requirement 16: Provide HTTP API Integration

**User Story:** As a developer, I want to integrate Excel parsing with the existing suites API, so that I can upload and process Excel files through the web interface.

#### Acceptance Criteria

1. THE Parser SHALL integrate with the existing suites controller to provide an Excel upload endpoint
2. WHEN an Excel file is uploaded, THE Parser SHALL process it and return the extracted tenant data
3. WHEN parsing succeeds, THE Parser SHALL store the extracted data using the existing suites service
4. WHEN parsing fails, THE Parser SHALL return appropriate HTTP error status with error details

### Requirement 17: Handle Excel Parsing Errors

**User Story:** As a financial analyst, I want the system to handle Excel parsing errors gracefully, so that I receive clear feedback when files cannot be processed.

#### Acceptance Criteria

1. WHEN an Excel file is corrupted or unreadable, THE Parser SHALL return a descriptive error message
2. WHEN required sections are missing from the Excel file, THE Parser SHALL log specific missing section errors
3. WHEN data extraction fails for specific tenants, THE Parser SHALL continue processing other tenants and report partial results
4. WHEN no tenant data is found, THE Parser SHALL return an error indicating no extractable data was found
